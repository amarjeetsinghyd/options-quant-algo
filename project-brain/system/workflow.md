# Scientific Engineering Lifecycle

Every engineering execution must follow this deterministic sequence. No stage may be skipped. Observation always precedes planning, and knowledge update is always final.

1. **Problem:** Identify the engineering objective or user request.
2. **Relevant Observation:** Consult the Canonical Observation Dataset (if applicable) for market reality ground truth.
3. **Relevant Institutional Knowledge:** Read `docs/institution/` to ensure constitutional alignment.
4. **Relevant Architecture:** Read `docs/architecture/` for systemic boundaries.
5. **Planning:** Generate a structural blueprint (See `planner.md`).
6. **Execution:** Modify source code or graphs.
7. **Validation:** Static validation (lint, test) and replay confirmation.
8. **Scientific Review:** Peer review against `standards/` and `reviews/`.
9. **Knowledge Update:** Update `project-brain/memory/` runtime state and, if scientifically validated, propose updates to `docs/institution/` or `docs/research/`.
